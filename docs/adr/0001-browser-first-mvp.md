# ADR 0001: Browser-first MVP

## Решение

Первая версия приложения реализуется как браузерный `1:1 WebRTC` сервис без `Flutter`, без нативных клиентов и без обязательной базы данных.

## Контекст

Целевой пользовательский сценарий:

1. Пользователь 1 создает сессию на сайте.
2. Сразу попадает в экран видеосвязи и видит локальное видео.
3. Копирует ссылку на сессию одной кнопкой.
4. Пользователь 2 открывает ссылку и сразу присоединяется.
5. После выхода обоих сессия завершается.

## Последствия

- уменьшается объем инфраструктуры и сложность первой версии;
- локальный и production запуск упрощаются;
- сессии хранятся в памяти сервера и подходят для одного экземпляра приложения;
- при масштабировании позже потребуется общий state store, например `Redis`.
