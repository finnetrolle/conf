# Задача: Заменить сценарий "скопируй ссылку" на полноценное приглашение в звонок

## Приоритет

`P0` — приглашение в один шаг напрямую влияет на то, попадет ли родитель в звонок без посторонней помощи.

## Контекст

Сейчас ключевое действие приглашения сводится к копированию ссылки в буфер обмена.

Для сценария быстрых звонков родителям этого недостаточно: пользователю нужен простой путь "отправить приглашение", а не техническое действие "скопировать URL".

## Проблема

Копирование ссылки требует дополнительных шагов вне продукта. Пользователь должен сам решить, куда вставить ссылку и как объяснить, что с ней делать. Это повышает трение и часто ломает звонок на самом простом этапе.

## Что нужно сделать

- Добавить сценарий "Отправить приглашение" как отдельное основное действие.
- Поддержать `Web Share API` на мобильных устройствах.
- Добавить быстрые варианты отправки в `WhatsApp`, `Telegram`, `SMS` и другие типовые каналы.
- Добавить QR-код для подключения с другого устройства.
- Подготовить понятный текст приглашения, например "Анна приглашает вас на видеозвонок".

## Почему это важно

Если приглашение можно отправить в один шаг, вероятность успешного подключения заметно выше. Для семейного звонка важнее не наличие URL, а простота передачи и понятность для получателя.

## Критерии готовности

- В интерфейсе есть отдельная кнопка "Отправить приглашение".
- На мобильных устройствах вызывается системное окно шаринга, если браузер это поддерживает.
- В продукте предусмотрен хотя бы один fallback-способ отправки, если системный шаринг недоступен.
- Получатель приглашения видит человеко-понятный текст, а не только голую ссылку.
- Пользователь может подключить второе устройство через QR-код без ручного ввода ссылки.
